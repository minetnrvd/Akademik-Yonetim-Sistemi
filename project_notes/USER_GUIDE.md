# User Guide

## 1. Purpose
This document explains how students, teachers, and admins use the QR Attendance system.

## 2. Roles
- Student: joins classes, marks attendance, tracks absence.
- Teacher: creates classes and sessions, views class/session details, reports.
- Admin: manages users, lock/role operations, permission and security views.

## 3. Login and Account
1. Open the login page.
2. Enter email and password.
3. Select "Remember Me" if needed.
4. If login fails repeatedly, account protection may temporarily lock access.

## 4. Student Flow
1. Open Absence page.
2. Join class from available class list.
3. Use attendance link/QR when an active session exists.
4. Check attendance history and report summary.

## 5. Teacher Flow
1. Create class.
2. Start attendance session for a week.
3. Show QR and let students scan.
4. Finish session and review session/class details.
5. Review reports for attendance trends.

## 6. Admin Flow
1. Open user inventory.
2. Update role or lock state when needed.
3. Use class assignment screen to reassign teachers.
4. Review permission and operation audit pages.
5. Use notifications page for test or broadcast emails.

## 7. Common Problems
- "Session expired": log in again.
- "Too many login attempts": wait for the limit window.
- "Account locked": contact admin.
- "Invalid QR": verify student is enrolled in the class.

## 8. Security Tips
- Use strong passwords.
- Log out on shared devices.
- Do not share attendance links publicly.
